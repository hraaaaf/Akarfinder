# Dynamic Search Profile V2 — #19D

## Doctrine

A SearchProfile is not a persona. It is a versioned, evolving representation of one concrete property-search project.

It contains:
- objective;
- intended uses;
- useful personal context when relevant;
- preferred/excluded locations and anchors;
- budget and flexibility;
- absolute property constraints;
- preferences;
- priorities;
- tolerances and accepted compromises.

Labels such as family, investor, MRE, student or retiree may exist as context/use signals but never select a hidden preset weight bundle.

## Evidence priority

Profile signals record source and confidence:
- `explicit` — strongest;
- `companion_derived` — structured conclusion from explicit answers;
- `behavioral_inference` — weak signal only.

Behavioral inference never creates an absolute constraint and is down-weighted compared with explicit preferences.

## Readiness

Search readiness requires the minimum project frame: objective, intended use, location and objective-appropriate budget. Preferences may remain unknown and can be learned progressively.

## Relationship to Neighborhood Intelligence

Neighborhood preferences use the same semantic dimensions as Neighborhood Intelligence V2: calmness, family fit, coastal lifestyle, tourism intensity, walkability, transit, investment use signals, etc.

A preference is not evidence that a neighborhood possesses the attribute. #19E compares SearchProfile preferences against available Neighborhood Intelligence evidence.

## Persistence boundary

This module is the deterministic profile contract/engine. Multi-project account persistence, history, favorites and continuity belong to #19H.
